## 需求分析

### 1. 需求概述

本软件项目管理平台是一款基于 Web 的前后端分离项目管理系统，面向小型团队与课程项目组。系统核心目标是支持多项目并行管理、项目进度规划、任务跟踪、敏捷协作与团队沟通，覆盖项目全生命周期，从需求、计划、执行到交付，实现信息集中、进度可控、协作高效。

### 2. 功能需求

#### 2.1 多项目管理

- 支持项目列表展示与项目收藏
- 提供自定义仪表盘，展示个人或项目概览
- 支持项目公开/私密状态设置
- 支持按多维属性筛选项目，并可分享筛选结果
- 支持项目组合与项目集管理，协同调整优先级与整体进度
- 支持自定义项目生命周期阶段与阶段关口
- 提供项目模板功能，可快速复用项目配置
- 支持项目变更日志，跟踪工作包、属性和自定义字段变更
- 支持跨项目自定义筛选视图共享给团队成员

#### 2.2 项目规划与进度管理

- 支持甘特图可视化规划，展示任务、里程碑与交付逻辑
- 支持工作包拆解为阶段、任务、里程碑、缺陷等粒度
- 支持设置任务关联关系：前驱/后继、阻塞、父子层级
- 提供自动与手动排程模式
- 支持自定义工作日历与休假设置
- 支持项目基线对比，查看进度偏差与变更影响
- 提供日历视图展示任务，并支持 iCalendar 格式订阅与提醒
- 支持团队人力规划，查看成员任务分配与时间安排

#### 2.3 任务管理与问题跟踪

- 支持动态任务列表筛选与分类展示
- 管理待办、进行中、已完成等任务状态
- 支持为任务分配负责人、受派者与观察者
- 提供任务变更通知与消息提醒
- 支持任务附件上传，实现任务与文件一体化管理
- 支持任务状态、优先级、截止日期高亮显示
- 支持工作计划导出为多种格式

#### 2.4 敏捷看板与 Scrum 管理

- 提供灵活配置的看板界面
- 支持团队看板、版本看板、子项目看板等多种视图
- 支持可视化展示工作分解结构（WBS）
- 支持产品待办、冲刺待办事项管理
- 提供冲刺看板和障碍管理，适配 Scrum 迭代流程

#### 2.5 时间、成本与预算管理

- 支持按日/周记录工作包工时
- 支持工时历史查询与修改
- 提供工时统计与成本分析报表
- 支持成员工时费率与人工成本核算
- 支持自定义单位成本分摊到工作包
- 提供项目预算编制与预算执行监控功能

#### 2.6 团队协作管理

- 显示项目活动流与动态同步
- 支持在线文档共享与协作编辑
- 提供会议议程管理与会议记录归档
- 支持项目新闻发布与动态通知
- 支持在任务中评论并 @ 成员
- 提供项目维基与团队论坛功能

#### 2.7 产品路线图与发布规划

- 提供可视化产品路线图展示
- 支持版本、任务与里程碑时间线展示
- 支持与 GitHub 集成，打通项目管理与研发流程

#### 2.8 工作流程与定制化

- 支持自动化项目启动与审批工作流
- 支持自定义工作包状态流转规则
- 支持为项目与任务属性添加帮助说明
- 支持自定义扩展字段与属性配置
- 提供细粒度角色、权限与用户组管理
- 支持多语言界面切换

#### 2.9 安全管理

- 用户密码加密存储，保障账户安全
- 支持双因素认证（手机/邮箱验证）
- 支持文件上传安全校验与病毒检查
- 外部链接跳转显示安全提示

### 3. 非功能需求

#### 3.1 性能需求

- 页面操作响应时间 ≤ 2 秒
- 数据查询与报表生成 ≤ 5 秒
- 支持至少 20 人同时在线操作，无明显卡顿
- 列表与图表采用分页加载，避免页面卡顿

#### 3.2 易用性需求

- 界面简洁直观，符合常规项目管理工具使用习惯
- 支持可视化图表与看板展示，降低操作门槛
- 提供清晰操作提示与新手引导

#### 3.3 安全性需求

- 基于角色的权限控制，不同角色仅访问授权功能
- 用户数据加密存储，支持文件安全校验
- 操作日志可追溯，保障数据审计

#### 3.4 兼容性需求

- 支持主流浏览器访问：Chrome、Edge、Firefox
- 适配 PC 端常规屏幕分辨率
- 支持前后端跨域请求正常处理

#### 3.5 可维护性需求

- 代码结构清晰，遵循开发规范
- 数据库表结构设计合理，支持备份与恢复
- 功能模块独立，便于后续扩展与迭代

### 4. 约束条件

- 采用前后端分离开发模式
- 技术栈限定：前端 Vue/React，后端 SpringBoot/Django，数据库 MySQL
- 适用于课程设计场景，聚焦核心功能，无需复杂企业级定制
- 部署环境为本地或轻量云 Web 服务器
