## 需求分析
1. 需求概述

本软件项目管理平台是基于Web的前后端分离项目管理系统，面向小型团队、课程项目组，满足多项目并行管理、项目进度规划、任务跟踪、敏捷协作、团队沟通等核心管理需求，覆盖项目全生命周期管理流程，解决传统项目管理中信息分散、进度难控、协作低效的问题。

2. 功能需求

2.1 多项目管理

• 支持个性化项目列表展示，提供项目收藏、自定义仪表盘功能，可设置项目公开/私密状态

• 支持按项目属性筛选列表，可将筛选后的项目总览分享给指定用户或小组

• 支持项目组合、项目集管理，可协调项目优先级、监控整体进展

• 支持自定义项目概览页面，通过图表、小部件可视化项目状态与进度

• 支持自定义项目生命周期阶段与阶段关口，跟踪项目推进流程

• 提供项目模板功能，可复用模板快速创建新项目

• 搭建个人专属仪表盘，聚合个人关联项目信息，支持个性化配置

• 全程记录项目变动日志，跟踪工作包、项目属性及自定义字段变更

• 支持跨项目自定义筛选视图，可共享统一视图给团队成员

2.2 项目规划与进度管理

• 支持甘特图可视化规划，展示工作包、里程碑时间节点与交付逻辑

• 支持工作包拆解（阶段、任务、里程碑、缺陷等），可分配负责人及设置属性

• 支持配置工作包关联关系：前驱/后继、阻塞、父子层级等

• 提供自动、手动两种排程模式，支持自定义工作日历

• 支持项目基线对比，查看工作包变更与进度偏差

• 提供日历视图展示任务，支持iCalendar格式订阅，配置任务日期提醒

• 支持团队人力规划，查看成员任务分配与时间安排

2.3 任务管理与问题跟踪

• 生成动态可筛选任务列表，分类管理待办、进行中、已完成任务

• 支持为工作包分配受派者、责任人、观察者，观察者实时接收变更通知

• 按预设规则自动生成标准化工作包主题，统一命名规范

• 提供实时系统通知，支持自定义邮件通知规则

• 支持工作包跨用户、跨项目分享，兼顾数据私密与信息同步

• 支持工作包附件上传，实现任务与文件一体化管理

• 任务状态、优先级、截止日期高亮提示，突出重点待办任务

• 支持工作计划多格式导出

2.4 敏捷看板与Scrum管理

• 支持自定义基础看板，灵活配置卡片与栏目

• 提供团队看板、版本看板、子项目看板，适配不同敏捷协作场景

• 支持通过看板可视化展示工作分解结构（WBS）

• 支持产品待办、冲刺待办事项管理，可进行优先级排序

• 搭建冲刺专用看板，记录迭代任务与障碍，适配Scrum协作模式

2.5 时间、成本与预算管理

• 支持按日/周记录工作包工时，可查询、修改历史工时记录

• 支持自定义生成工时统计、成本分析报表

• 可配置团队成员工时费率，自动核算项目劳动力成本

• 支持自定义单位成本，可分摊至对应工作包

• 提供项目预算编制功能，实时监控预算执行状态

2.6 团队协作管理

• 展示项目活动流，实时同步项目变更与动态

• 支持团队文档在线共享与协作编辑

• 提供会议议程管理、会议记录归档功能

• 支持项目新闻发布与动态推送

• 支持评论@指定成员，实现定向通知

• 搭建项目维基知识库，统一管理项目文档与规范

• 提供团队论坛，支持项目话题讨论与交流

2.7 产品路线图与发布规划

• 可视化展示产品路线图，呈现版本与项目进展

• 以时间轴形式展示产品功能、任务与里程碑

• 支持与GitHub集成，打通项目管理与研发流程

2.8 工作流程与定制化

• 支持自动化项目启动工作流配置

• 自定义工作包状态流转规则

• 为项目/工作包属性配置帮助说明文本

• 支持自定义扩展项目、工作包字段

• 提供细粒度角色、权限与用户组管理

• 支持多语言切换

2.9 安全管理

• 用户密码加密存储，支持安全认证

• 提供双因素认证功能（手机/邮箱验证）

• 上传文件自动进行病毒扫描

• 外部链接跳转时展示安全警告提示

3. 非功能需求

3.1 性能需求

• 系统响应速度：普通页面操作响应时间≤2秒，数据查询/报表生成≤5秒

• 并发支持：支持至少20人同时在线操作，无明显卡顿

• 数据加载：列表、图表数据分页加载，避免页面卡顿

3.2 易用性需求

• 界面简洁直观，操作流程符合常规项目管理工具使用习惯

• 支持图表、看板等可视化展示，降低操作门槛

• 提供清晰的提示与引导，新手可快速上手核心功能

3.3 安全性需求

• 基于角色的权限管控，不同角色仅可访问对应功能

• 用户数据加密存储，文件上传安全校验

• 操作日志可追溯，保障项目数据安全

3.4 兼容性需求

• 支持主流浏览器（Chrome、Edge、Firefox）访问，适配PC端操作

• 前后端接口兼容，支持跨域请求正常处理

3.5 可维护性需求

• 代码结构清晰，遵循开发规范，便于后续功能迭代与bug修复

• 数据库表结构设计合理，支持数据备份与恢复

• 支持功能模块独立扩展，不影响核心流程

4. 约束条件

• 采用前后端分离开发模式，技术栈限定为前端Vue/React、后端SpringBoot/Django、数据库MySQL

• 适用于课程设计场景，功能聚焦核心需求，无需复杂企业级定制

• 部署环境为常规Web服务器，适配本地/轻量云服务器部署
