# 用例交互场景

## 1. 登录与权限验证
- 参与者：所有用户
- 前置条件：用户已注册并拥有账号
- 主流程：
  1. 用户进入登录页面。
  2. 输入用户名和密码。
  3. 系统验证身份并跳转到权限允许的首页。
- 备用流程：
  1. 用户输入错误密码。
  2. 系统提示“用户名或密码错误”，允许重新输入。
- 后置条件：用户成功登录并进入个人仪表盘。

## 2. 创建项目
- 参与者：项目经理
- 前置条件：项目经理已登录系统并具有项目创建权限
- 主流程：
  1. 点击“创建项目”按钮。
  2. 输入项目名称、描述、开始日期、结束日期和项目成员。
  3. 选择项目可见性（公开/私密）。
  4. 点击“保存”创建项目。
  5. 系统显示项目概览页面，并在项目列表中添加新项目。
- 异常流程：
  1. 项目名称为空或重复时，系统显示错误提示，要求补充或修改。
- 后置条件：项目创建成功，成员收到项目加入通知。

## 3. 创建任务
- 参与者：项目成员
- 前置条件：用户已进入具体项目页面并有任务创建权限
- 主流程：
  1. 在项目页面选择“任务列表”或“看板”视图。
  2. 点击“新增任务”。
  3. 填写任务标题、说明、负责人、优先级、开始日期和截止日期。
  4. 设置任务所属阶段、前驱任务和附件（可选）。
  5. 点击“保存”。
  6. 系统将任务添加到列表/看板，并通知负责人。
- 异常流程：
  1. 负责人未选择时，系统提示必须指定负责人。
- 后置条件：任务成功创建，进入待办或指定默认状态列。

## 4. 编辑任务详细信息
- 参与者：任务负责人、项目经理
- 前置条件：任务已存在并且用户有编辑权限
- 主流程：
  1. 在任务列表或看板中点击任务卡片。
  2. 编辑任务标题、描述、责任人、截止日期或关联关系。
  3. 保存更改。
  4. 系统记录变更日志并通知相关成员。
- 备用流程：
  1. 任务因权限受限无法编辑，系统提示“无权限编辑”。
- 后置条件：任务信息更新成功，变更日志可查询。

## 5. 更新任务状态
- 参与者：团队成员
- 前置条件：任务已创建且当前用户有状态更新权限
- 主流程：
  1. 在看板或任务详情页选择任务。
  2. 将任务卡片拖动到目标状态列（如“进行中”、“已完成”）。
  3. 系统保存状态变更，并更新任务进度条。
  4. 系统向观察者和负责人发送通知。
- 异常流程：
  1. 任务当前状态受限时，系统提示“当前状态无法直接跳转”，并提供可用状态路径。
- 后置条件：任务状态更新成功，团队成员可见当前进度。

## 6. 查看甘特图与进度
- 参与者：项目经理、产品负责人
- 前置条件：项目中已创建任务和里程碑
- 主流程：
  1. 进入项目甘特图页面。
  2. 系统显示任务时间线、里程碑、前驱/后继关系和关键路径。
  3. 用户可以放大、缩小时间范围并筛选任务。
- 备用流程：
  1. 如果项目无任务，系统显示“当前项目暂无任务，请先创建任务”。
- 后置条件：用户可以根据甘特图调整任务排期。

## 7. 导出项目报告
- 参与者：项目经理、测试负责人
- 前置条件：项目已创建且有任务数据
- 主流程：
  1. 在项目概览或报表页面选择“导出报告”。
  2. 选择导出类型（任务列表、工时统计、甘特图、项目进度）。
  3. 系统生成文件并提供下载链接。
- 后置条件：用户下载报告文件用于展示或存档。
