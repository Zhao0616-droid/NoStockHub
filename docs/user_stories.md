# 用户故事

## 1. 项目与角色管理
- 作为项目经理，我希望创建、编辑和关闭多个项目，以便统一管理团队工作并监督总体进度。
- 作为团队成员，我希望查看我参与的项目、任务和里程碑，以便清楚自己的工作内容和时间安排。
- 作为系统管理员，我希望配置不同角色的权限，以便保障信息安全并让每个用户只看到应有功能。

## 2. 任务与进度规划
- 作为任务负责人，我希望创建任务、设置执行人、截止日期和任务依赖关系，以便明确工作顺序和责任人。
- 作为产品负责人，我希望在甘特图中查看各项目的里程碑和计划进度，以便及时发现延期并调整资源分配。
- 作为团队成员，我希望查看任务的历史更新和变更日志，以便了解任务状态变化原因。

## 3. 看板与敏捷协作
- 作为开发人员，我希望通过看板拖动任务卡片来更新任务状态，以便团队协作更灵活。
- 作为Scrum Master，我希望创建冲刺看板和版本看板，以便管理迭代目标、待办事项和障碍。
- 作为项目经理，我希望把任务分组到不同看板栏目，以便按优先级、阶段和责任人展示工作进展。

## 4. 团队沟通与通知
- 作为团队成员，我希望在任务中留言、@ 指定成员，以便快速沟通问题和进度。
- 作为项目经理，我希望开启邮件/系统通知规则，以便关键任务变更时及时提醒相关人员。
- 作为团队成员，我希望收到任务截止前提醒，以便提前准备并避免延期。

## 5. 数据与报表
- 作为项目经理，我希望导出项目状态报告和工时统计，以便向导师或客户展示项目进展。
- 作为财务负责人，我希望查看成员工时汇总和成本分析，以便评估项目预算执行情况。
- 作为用户，我希望导出工作计划为多种格式，以便离线查看和分享。
